# Beauty Pageant App

Beauty Pageant App is a local-first scoring system for pageants and similar events. It includes an admin dashboard, judge login, real-time leaderboard updates, contestant image uploads, and a separate elimination page for round cuts.

The app runs on Express with LowDB JSON storage and serves a vanilla HTML/CSS/JS frontend. UI assets are bundled locally into `public/vendor`, so the interface stays consistent even on LAN or hotspot-only setups.

## Features

- Admin dashboard for contestants, judges, categories, password changes, and full reset
- Judge login and score entry per category
- Real-time leaderboard updates with Socket.IO
- Separate elimination page with:
  - ranking view
  - top 10 / below top 10 filters
  - bulk `Eliminate` action
  - bulk `Proceed To Next Round` action
  - restore eliminated contestants
- Optional contestant photo uploads stored in `uploads/`
- Local JSON persistence in `db.json`
- Local frontend asset build for Bootstrap, Font Awesome, and fonts

## Stack

- Node.js
- Express
- LowDB
- Socket.IO
- Express Session
- Multer
- Vanilla JavaScript
- Bootstrap 5
- Font Awesome

## Pages

- Judge portal: `http://localhost:3000/`
- Admin dashboard: `http://localhost:3000/admin.html`
- Elimination page: `http://localhost:3000/elimination.html`
- Judge score sheet: `http://localhost:3000/judge.html`

## Requirements

- Node.js installed in Windows at `C:\Program Files\nodejs\node.exe`
  - recommended for the easiest firewall/LAN behavior
- Or use the bundled Node copy already included in the project

## Setup

1. Install dependencies:

```bash
npm install
```

2. Build local frontend assets:

```bash
npm run build
```

This copies Bootstrap, Font Awesome, and the project fonts into `public/vendor/`.

## Start The App

### Windows launcher

Use:

```bat
tabulator.bat
```

What it does:

- prefers the system Node.js install
- falls back to the bundled Node copy if needed
- checks whether port `3000` is already being used
- restarts the previous Beauty Pageant App instance cleanly if it is already running

### Standard Node start

You can also run:

```bash
npm start
```

## Default Admin Login

- Username: `admin`
- Password: `password`

Change it after first login.

## Environment Variables

The current app uses:

- `PORT`
- `SESSION_SECRET`

Example:

```env
PORT=3000
SESSION_SECRET=change-this-secret
```

Note:

- The app currently uses LowDB and `db.json`.
- Old `DB_*` values in `.env` are not used by the current setup.

## Data And Important Files

- `server.js` - Express server, API routes, sessions, Socket.IO, startup
- `db.json` - app data
- `uploads/` - contestant images
- `public/` - HTML, CSS, JS, and built vendor assets
- `public/vendor/` - locally bundled UI assets generated by `npm run build`
- `scripts/build-assets.js` - copies frontend dependencies into `public/vendor`
- `tabulator.bat` - Windows launcher

## LAN And Hotspot Use

When the server starts, it prints:

- `Local: http://localhost:3000`
- `LAN: http://<your-ip>:3000`

Use the `LAN` address on other devices connected to the same network.

Notes:

- If you change Wi-Fi or hotspot, the LAN IP usually changes.
- If you update frontend assets, hard-refresh the browser once.
- Some phones cannot access tethered clients when that same phone is acting as the hotspot host. In that case:
  - use another device connected to the hotspot
  - or use the laptop as the hotspot host instead

## Notes

- Leaderboard updates are pushed in real time with Socket.IO.
- Only active contestants appear in judge scoring and leaderboard views.
- Category weights should total `100%`.
- Uploaded files are limited to image types and `5 MB`.
- The app is designed for local/LAN use and does not require an internet connection after `npm install` and `npm run build`.
